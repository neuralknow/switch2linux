# Glossary

New to Linux? Some of the words you'll come across might seem confusing at first. Here's a plain-English guide to the most common terms. 📖

---

## A

**APT** — *Advanced Package Tool*. The command-line program used to install, update, and remove software on Ubuntu and Pop!_OS. Example: `sudo apt install firefox`.

---

## B

**Bash** — The default command-line shell on most Linux systems. When you open a Terminal, you're usually running Bash. It lets you type commands to control your computer.

**Boot** — The process of starting your computer. "Booting from USB" means starting your computer from a USB drive instead of the hard drive.

**Bootloader** — A small program that runs when your computer starts and lets you choose which operating system to boot into (e.g. GRUB, which manages dual-boot setups).

---

## C

**CLI** — *Command Line Interface*. Controlling your computer by typing text commands, instead of clicking with a mouse. See also: Terminal.

**CUPS** — The printing system used by Linux. Handles communication between your computer and printers.

---

## D

**Desktop Environment (DE)** — The visual interface of Linux — the taskbar, windows, icons, and system menus. GNOME is the most common one used by Ubuntu and Pop!_OS.

**Distro** — Short for *distribution*. A packaged version of Linux with a specific set of software and settings. Ubuntu, Pop!_OS, and Linux Mint are all distros.

**Dual Boot** — Having two operating systems (e.g. Windows and Linux) installed on the same computer, choosing which to use at startup.

---

## F

**Flatpak** — A universal package format that lets you install apps on any Linux distro, regardless of which one you're running.

---

## G

**GNOME** — The most popular desktop environment for Linux. Clean, modern, and used by Ubuntu and Pop!_OS.

**GRUB** — *Grand Unified Bootloader*. The program that appears when you start a dual-boot computer, letting you choose between Linux and Windows.

**GUI** — *Graphical User Interface*. The visual part of an operating system — windows, buttons, icons. The opposite of CLI.

---

## I

**ISO** — A file that contains an exact copy of a disc (like a DVD). You download an ISO file of Linux and put it on a USB drive to install from.

---

## K

**Kernel** — The core of the Linux operating system. It manages communication between your software and hardware. You rarely interact with it directly.

---

## L

**Live Environment** — A fully working version of Linux that runs directly from a USB drive without installing anything. Great for testing before you commit.

**LTS** — *Long-Term Support*. A version of Ubuntu (or other distros) that receives security updates and fixes for 5 years, making it more stable for everyday use.

---

## M

**Mount** — Making a drive or storage device accessible to the file system. When you plug in a USB drive, Linux "mounts" it automatically so you can access its files.

---

## O

**Open Source** — Software whose source code is publicly available for anyone to view, modify, and share. Linux, Firefox, and LibreOffice are all open source.

---

## P

**Package** — A bundle containing an app and everything it needs to run. Installing software on Linux is done by installing packages.

**Package Manager** — A tool that handles installing, updating, and removing packages. APT is the package manager used by Ubuntu and Pop!_OS.

**Partition** — A section of a hard drive. When you install Linux alongside Windows, the installer creates a new partition for Linux.

**PPA** — *Personal Package Archive*. A repository maintained by a developer or community to distribute software not in the official Ubuntu repository.

**Proton** — A compatibility layer developed by Valve (Steam) that allows many Windows games to run on Linux.

---

## R

**Repository (Repo)** — A server containing a collection of software packages. When you run `apt update`, Linux checks the repository for updates. Think of it as a trusted app store in the background.

**Root** — The highest level of the Linux file system (`/`). Also refers to the "superuser" or administrator account. `sudo` gives you temporary root privileges.

---

## S

**Shell** — A program that interprets your typed commands. Bash is the most common shell.

**Snap** — A universal package format created by Canonical (makers of Ubuntu). Similar to Flatpak.

**sudo** — *Superuser Do*. A command prefix that gives you administrator privileges for a single command. Example: `sudo apt install vlc`.

**System76** — The company that makes Pop!_OS. They also build Linux laptops and desktops.

---

## T

**Terminal** — A text-based interface for entering commands. On Ubuntu, open it by pressing Ctrl+Alt+T or searching "Terminal" in the app launcher.

**Tux** — The official mascot of Linux — a friendly penguin! 🐧

---

## U

**Ubuntu** — A popular, beginner-friendly Linux distribution maintained by Canonical. Based on Debian Linux.

---

## W

**Wine** — A compatibility layer that allows some Windows programs to run on Linux. Not needed for Steam games (which use Proton), but useful for other Windows software.
